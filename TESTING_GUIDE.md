# 功能测试指南

## 🎯 快速测试流程

### 1. 启动应用

确保前端和后端都在运行：

```bash
# 终端 1 - 后端
cd backend && npm run dev

# 终端 2 - 前端
cd frontend && npm run dev
```

访问: http://localhost:3000

### 2. 测试添加药品

1. 点击右上角"添加药品"按钮
2. 填写信息：
   - 药品名称: `阿司匹林`
   - 剂量: `1片`
   - 服药时间: `09:00`
   - 选择颜色: 青柠
3. 点击"添加"
4. 应该看到药品卡片出现在列表中

**测试多个药品**:
- 添加 3-5 个不同药品
- 使用不同颜色主题
- 设置不同服药时间

### 3. 测试拍照记录

1. 点击任意药品卡片上的"拍照记录"按钮
2. 允许浏览器访问摄像头
3. 点击"拍照"按钮
4. 查看预览
5. 点击"确认"
6. 应该看到成功提示："✅ 服药记录已保存"

### 4. 测试进度显示

1. 记录几个药品后
2. 查看左侧的"今日进度"卡片
3. 应该显示：
   - 环形进度图
   - 百分比数字
   - "已完成 X / Y 个药品"

### 5. 测试历史记录

1. 查看右侧的"最近7天记录"卡片
2. 应该看到：
   - 拍摄的照片
   - 药品名称
   - 服药时间
   - 状态图标（按时/延迟）

### 6. 测试编辑功能

1. 点击药品卡片上的"编辑"按钮（铅笔图标）
2. 修改药品信息
3. 点击"更新"
4. 应该看到更新后的信息

### 7. 测试删除功能

1. 点击药品卡片上的"删除"按钮（垃圾桶图标）
2. 确认删除
3. 药品应该从列表中消失

## 🐛 常见问题

### Q: 拍照功能无法使用
A: 检查浏览器是否允许摄像头权限，在浏览器设置中允许访问摄像头

### Q: 进度不更新
A: 刷新页面，或检查是否有服药记录

### Q: 记录不显示
A: 检查浏览器控制台是否有错误，确认数据已保存到 IndexedDB

### Q: 样式显示异常
A: 确认 Tailwind CSS 已正确加载，检查浏览器控制台
